# Project-C 주간 수렴 리포트 — 2026-W15

2026년 4월 둘째 주, Project-C R&D와 직결되는 6편의 논문·구현을 수렴한다. 이번 주의 공통 주제는 하나로 압축된다: **비디오 인페인팅 스택이 DiT 중심으로 재편되면서, 캐릭터 교체의 각 단계(마스크 추론 → 객체 제거 → 아이덴티티 전이 → 최종 합성)가 동시에 DiT-native 구현을 갖추기 시작했다.** 지난주(W14)가 파이프라인 블록의 "출현"이었다면, 이번 주는 각 블록의 **실전 연결 가능성**이 확인된 주다.

---

## 논문별 핵심 요약

### 1. MoCha — End-to-End 비디오 캐릭터 교체 (CVPR 2026)

**핵심 기여:** 단 한 프레임의 마스크만으로 전체 비디오 캐릭터를 교체하는 **최초의 end-to-end 프레임워크**. 기존 방법이 매 프레임 세그멘테이션 + 명시적 구조 가이드를 요구한 것과 달리, Condition-Aware RoPE로 비디오·마스크·레퍼런스를 단일 토큰 스트림으로 통합하고 RL 기반 후학습(facial reward)으로 아이덴티티 보존을 강화한다.
**핵심 지표:** CVPR 2026 채택; 코드·가중치 공개(GitHub + HuggingFace).

---

### 2. ReplaceAnyone — DiT/Wan2.1 호환 포즈 구동 캐릭터 교체

**핵심 기여:** 참조 이미지에서 4종 신호(외형·포즈·모션·배경)를 동시에 추출하는 **Enriched Visual Guidance**와 하이브리드 인페인팅 인코더로 포즈 보존과 어피런스 충실도를 동시 최적화. 전통 3D-UNet과 DiT(Wan2.1) 백본을 모두 지원한다.
**핵심 지표:** Wan2.1 DiT 백본 지원 확장(v2); 단일 프레임워크에서 교체·삽입 동시 처리.

---

### 3. LanPaint v1.5 — 학습-프리 범용 인페인팅 샘플러 (TMLR)

**핵심 기여:** 모델 재학습 없이 **임의의 디퓨전 모델에 인페인팅 능력을 부여**하는 범용 샘플러. 디노이징 스텝마다 마스크 영역을 반복 정제하는 "Think Mode"(점근적 정확 조건부 샘플링)로 완전히 새로운 콘텐츠를 생성하며 Wan2.2 비디오 포함 12+ 모델을 지원한다.
**핵심 지표:** Wan2.2 + 880×480 + 40프레임 기준 39.8 GB VRAM, 추론 5.5분; MIT 라이선스; ComfyUI 네이티브.

---

### 4. EraserDiT — 1080p 고속 비디오 인페인팅 DiT

**핵심 기여:** LTX-Video를 백본으로 하는 DiT 아키텍처로 **대형 마스크 영역·시간 일관성** 문제를 해결. 별도 가속 기법 없이 1080p 장편 비디오를 실용적 속도로 처리한다.
**핵심 지표:** 1080×1920 해상도 121프레임, A100 단일 GPU에서 **180초** 완료.

---

### 5. DiTPainter — 경량 From-Scratch DiT 비디오 인페인팅

**핵심 기여:** 사전학습 모델 없이 비디오 인페인팅 전용으로 **처음부터 훈련**한 소형 DiT. Flow Matching 채택으로 추론 스텝을 극단적으로 줄이면서 기존 알고리즘 대비 품질·시공간 일관성 모두 우세.
**핵심 지표:** **4~8 스텝**으로 만족스러운 품질 달성 (기존 DDPM 기반 50~100 스텝 대비 10× 이상 가속).

---

### 6. VOID — 물리 인식 비디오 객체·상호작용 제거 (Netflix)

**핵심 기여:** 객체 제거를 단순 인페인팅이 아닌 **반사실적 추론**("이 객체가 처음부터 없었다면?")으로 재정의. 충돌·그림자·반사 등 물리적 상호작용까지 반영하여 현실적인 제거 결과를 생성한다.
**핵심 지표:** Netflix 오픈소스; 코드·모델 즉시 사용 가능(GitHub + HuggingFace).

---

## 핵심 트렌드

### 1. DiT가 비디오 인페인팅의 새로운 기준 아키텍처로 확정되었다

이번 주 6편 중 4편(MoCha, ReplaceAnyone, EraserDiT, DiTPainter)이 DiT 또는 DiT-compatible 백본을 채택한다. LTX-Video, Wan2.1/2.2, 자체 설계 소형 DiT까지 — 다양한 구현 경로가 모두 DiT 방향을 향한다. Project-C 파이프라인의 **인페인팅 블록 교체 시 DiT-native 방식을 기본 선택지로 고정**할 근거가 이번 주로써 충분히 확보되었다.

### 2. 캐릭터 교체의 "상류 문제"가 end-to-end로 흡수되고 있다

MoCha는 세그멘테이션 마스크 추론, 구조 가이드 생성, 아이덴티티 전이를 모두 단일 모델 안에 통합했다. ReplaceAnyone은 포즈·모션·배경 신호를 하나의 인코더 패스에서 처리한다. 파이프라인 블록을 순차 연결하던 W14의 청사진과 달리, **이번 주부터는 "블록 수 최소화" 전략이 현실적 선택지**가 되었다. MoCha(단일 마스크 → 전체 교체) + LanPaint(학습-프리 후처리 인페인팅)의 2-블록 구성이 가장 빠른 PoC 경로로 부상한다.

### 3. 오픈소스 즉시 실험 가능한 모델이 3종 확보되었다

- **MoCha**: GitHub + HuggingFace 공개
- **VOID**: Netflix 오픈소스, HuggingFace 모델 제공
- **LanPaint**: MIT 라이선스, ComfyUI-Manager 원클릭 설치

EraserDiT·DiTPainter는 코드 미공개이나, LanPaint가 Wan2.2 백본에서 동등 기능을 즉시 대체할 수 있다. **이번 주 기준으로 Project-C PoC를 0원 VRAM 비용 없이 시작할 수 있는 오픈 스택이 완성된 상태**다.

## 기술 교차점

| 파이프라인 단계 | W14 최선 옵션 | W15 갱신 옵션 | 변화 |
|---|---|---|---|
| 캐릭터 세그멘테이션 | SAM 3.1 | SAM 3.1 (유지) | — |
| 객체 제거 / 배경 복원 | ProPainter류 | VOID + LanPaint | DiT 기반 + 물리 인식 |
| 아이덴티티 전이 | HiFi-Inpaint (코드 미공개) | MoCha (공개) | 즉시 실험 가능 |
